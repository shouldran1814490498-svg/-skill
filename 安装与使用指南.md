## 二、技能概览

```
┌────────────┐     ┌────────────┐     ┌────────────────┐
│  策略生成   │ ──→ │  达人筛选   │ ──→ │ 内容生成&审核   │
└────────────┘     └────────────┘     └────────────────┘
                          │
                          └──→ 比价核验（可选）
```

| 技能 | 一句话说明 | 触发命令 |
|------|-----------|----------|
| 策略生成 | 输入药品名 → 输出完整投放策略报告 | `/strategy` |
| 达人筛选 | 输入策略+KOL表+预算 → 输出筛选列表+看板 | `/kol-screen` |
| 内容生成&审核 | 输入达人表+策略 → 输出个性化笔记+合规审核 | `/xhs` |
| 比价核验 | 输入PO单+基准库 → 输出风险报告 | `/quote-audit` |
| 流水线编排 | 串联以上技能，自动传递数据 | `/pipeline` |

## 三、单独使用

### 1. 策略生成

生成某药品的完整社媒投放策略报告。

```
/strategy 乳果糖口服溶液
```

**输入**：药品名称（必填），品牌名/厂商（可选）

**输出**：`{药品名}_投放策略报告.md`，包含：
- 药品基础档案 & 合规红线
- 利益点拆解 & 社媒语言转化
- 竞品格局分析
- 达人矩阵 & 内容方向
- 投放节奏建议

---

### 2. 达人筛选

根据策略报告和 KOL 资源表，自动筛选达人、分配预算、生成可视化看板。

```
/kol-screen {策略报告} {KOL资源表} {蒲公英价格库} {预算} {头腰比} {渠道}
```

**示例**：
```
/kol-screen D:\data\策略报告.md D:\data\KOL资源表.xlsx D:\data\蒲公英报价.xlsx 100万 6:4 小红书
```

**输入**：

| 参数 | 说明 |
|------|------|
| 策略报告 | 来自策略生成的 .md 文件，或你自己准备的 brief |
| KOL资源表 | 含达人信息的表格（Joyspace链接 或 本地Excel） |
| 蒲公英价格库 | 蒲公英平台报价表（Joyspace链接 或 本地Excel） |
| 预算 | 总投放预算，如"100万" |
| 头腰比 | 头部:腰部预算分配，如"6:4" |
| 渠道 | 小红书 / 抖音 |

**输出**：
- `{药品名}_预算筛选_{渠道}.xlsx` — 筛选后的达人列表
- `{药品名}_投放看板.html` — 可视化看板（浏览器打开）
- `{药品名}_比价结果.csv` — 与蒲公英报价对比

---

### 3. 内容生成 & 审核

为每位达人生成个性化小红书笔记，自动执行 7 维合规审核。

```
/xhs {达人明细表} {投放策略报告}
```

**示例**：
```
/xhs D:\data\达人明细.xlsx D:\data\福元_乳果糖口服溶液_投放策略报告.md
/xhs D:\data\达人明细.csv D:\data\福元_乳果糖口服溶液_投放策略报告.md
```

**输入**：

| 参数 | 说明 |
|------|------|
| 达人明细表 | 22列标准格式的 .xlsx 或 .csv 文件 |
| 投放策略报告 | 来自策略生成的 .md 文件 |

**输出**：在原表基础上新增 23-29 列：

| 新增列 | 内容 |
|--------|------|
| 第23列 | 分配角度 |
| 第24列 | 笔记标题 |
| 第25列 | 笔记正文（原始版） |
| 第26列 | 话题标签 |
| 第27列 | 封面建议 |
| 第28列 | 合规审核结果（评分+问题+风险） |
| 第29列 | 修正后正文（合规通过则填"无需修改"） |

---

### 4. 比价核验

对比供应商 PO 报价与基准价格库，输出风险等级。

```
/quote-audit {PO报价单} {基准价格库}
```

**示例**：
```
/quote-audit D:\data\供应商报价.xlsx D:\data\蒲公英基准价.xlsx
```

**输入**：供应商PO单 + 基准价格（蒲公英/自定义），支持 Excel、CSV、在线文档

**输出**：结构化 Markdown 审核报告，含：
- 逐条比对结果（报价/基准价/比值/风险等级）
- 总体风险评级
- 异常项标记 & 建议动作

## 四、组合使用

你可以把任意几个技能串起来用。上一步的输出文件直接作为下一步的输入。

### 示例 1：策略 + 内容（跳过筛选，已有达人表）

```
# 第一步：生成策略
/strategy 乳果糖口服溶液

# 第二步：用策略 + 已有达人表生成内容
/xhs D:\data\达人明细.xlsx ./output/乳果糖口服溶液_投放策略报告.md
```

### 示例 2：策略 + 筛选（只需要达人列表，不需要内容）

```
# 第一步：生成策略
/strategy 乳果糖口服溶液

# 第二步：筛选达人
/kol-screen ./output/乳果糖口服溶液_投放策略报告.md D:\data\KOL.xlsx D:\data\蒲公英.xlsx 100万 6:4 小红书
```

### 示例 3：筛选 + 内容（已有策略报告）

```
# 第一步：筛选达人
/kol-screen D:\data\策略报告.md D:\data\KOL.xlsx D:\data\蒲公英.xlsx 80万 7:3 小红书

# 第二步：用筛选结果生成内容
/xhs ./output/乳果糖_预算筛选_小红书.xlsx D:\data\策略报告.md
```

### 示例 4：筛选 + 比价（聚焦价格审核）

```
# 第一步：筛选达人
/kol-screen D:\data\策略报告.md D:\data\KOL.xlsx D:\data\蒲公英.xlsx 100万 6:4 小红书

# 第二步：对筛选结果做深度比价
/quote-audit ./output/乳果糖_预算筛选_小红书.xlsx D:\data\蒲公英.xlsx
```

## 五、全链路一键执行

使用流水线编排，从药品名开始一路到最终内容输出，全程无需手动传递文件。

### 全链路执行

```
/pipeline full {药品名} {KOL资源表} {蒲公英价格库} {预算} {头腰比} {渠道}
```

**示例**：
```
/pipeline full 乳果糖口服溶液 D:\data\KOL资源表.xlsx D:\data\蒲公英报价.xlsx 100万 6:4 小红书
```

**执行过程**：
```
Step 1: 策略生成 ── 输出策略报告
         │
Step 2: 达人筛选 ── 输出筛选列表 + 看板
         │
Step 3: 内容生成&审核 ── 输出完整内容表
         │
      汇总报告
```

全程自动，你只需等待最终结果。

---

### 从中间步骤开始（已有部分产出）

```
# 已有策略报告，从筛选开始
/pipeline from-step2 D:\data\策略报告.md D:\data\KOL.xlsx D:\data\蒲公英.xlsx 100万 6:4 小红书

# 已有策略报告+达人表，从内容生成开始
/pipeline from-step3 D:\data\达人明细.xlsx D:\data\策略报告.md
```

---

### 自定义组合

```
# 只跑策略+内容（跳过筛选）
/pipeline combo step1+step3 乳果糖口服溶液

# 只跑策略+筛选（不生成内容）
/pipeline combo step1+step2 乳果糖口服溶液 D:\data\KOL.xlsx D:\data\蒲公英.xlsx 100万 6:4 小红书

# 只跑筛选+内容
/pipeline combo step2+step3 D:\data\策略报告.md D:\data\KOL.xlsx D:\data\蒲公英.xlsx 100万 6:4 小红书
```

## 七、常见问题

**Q: 安装后怎么更新？**
A: 自动更新。作者推送新版本后你会自动收到，无需操作。

**Q: 我只装了一个 skill，后面想加装其他的可以吗？**
A: 可以，随时执行对应的安装命令即可。也可以卸载单个再装全量包。

**Q: 达人表必须是 22 列吗？**
A: 是的，内容生成 skill 要求固定 22 列格式。如果你的表格列数不同，需要先调整格式。

**Q: 支持 CSV 还是 Excel？**
A: 都支持。`.xlsx` 和 `.csv`（自动检测编码 UTF-8/GBK）均可。

**Q: 策略报告一定要用策略生成 skill 产出的吗？**
A: 不一定。只要你的报告符合标准 Markdown 章节格式（药品档案、利益点拆解、投放策略框架等），就可以直接作为输入。

**Q: 全链路跑到一半失败了怎么办？**
A: 已完成步骤的输出会保留在 `./output/` 下，用 `/pipeline from-step2` 或 `from-step3` 从断点继续。

**Q: 我的 skill 版本和别人不一样怎么办？**
A: 所有人用同一个 URL 安装，指向同一个源仓库，版本始终一致。
